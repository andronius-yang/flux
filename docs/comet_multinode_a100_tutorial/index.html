The thesis

PAPER COMET overlaps MoE communication and computation by decomposing shared tensors, rescheduling independent tiles, and allocating GPU resources according to the target architecture.

CODE In this repository, Hopper V3 uses persistent warp-specialized GEMM for layer 0 and horizontally specialized compute/communication CTAs for layer 1.

DIFF The patch does not port those Hopper kernels wholesale. Layer 0 borrows node-aware NVSHMEM all-gather orchestration while retaining the Ampere V2 grouped GEMM. Layer 1 builds an A100-specific hierarchy: intra-node V2 rings, symmetric GPU staging, host-enqueued NVSHMEM put-with-signal transfers, then a GPU accumulation kernel.

The two data-dependency chains

CODE Layer 0 communicates the token rows before expert computation: all-gather → routing scatter → grouped GEMM. Layer 1 computes expert outputs before combining copies of each token: grouped GEMM → gather → top-k reduction → reduce-scatter. Those directions explain why M is the useful layer-0 scheduling axis and N is the useful layer-1 split axis.

First source walkthrough: architecture dispatch

examples/moe.py:148–186 · WORKTREE · abridgedPython / current stream
tp_env = flux.DistEnvTPWithEP(
    tp_group=TP_GROUP, nnodes=DIST_ENV.NNODES, ep_group=EP_GROUP)

if flux.util.get_arch() >= 90:
    self.flux_ag_op = flux.GemmGroupedV3AGScatter(...)
    self.flux_rs_op = flux.GemmGroupedV3GatherRS(...)
else:
    self.flux_ag_op = flux.GemmGroupedV2AGScatterOp(...)
    self.flux_rs_op = flux.GemmGroupedV2GatherRSOp(
        ..., nnodes=DIST_ENV.NNODES)

Execution. Python captures node count in the topology object, selects V3 only for compute capability 9.0+, and passes nnodes into both pre-SM90 operators.

Ownership and stream. These long-lived operator objects own communication buffers, peer mappings, events, and auxiliary streams. The calls themselves enter on PyTorch’s current CUDA stream.

Invariant. Architecture dispatch and topology dispatch are separate: SM80 chooses V2; nnodes > 1 then activates the worktree’s hierarchy inside V2.

Why it exists. Preserve architecture-native kernels while changing orchestration only where topology requires it.

Local glossary

CTA
CUDA cooperative thread array; a thread block.
EP / TP
Expert parallelism / tensor parallelism.
PE
NVSHMEM processing element, mapped here to a global rank.
PGAS
Partitioned global address space: symmetric objects share an offset across PEs, while ownership remains distributed.
Readiness flag
Not merely a Boolean: a publication contract saying the associated bytes are visible at the required scope.
V2 / V3
Flux implementation families for pre-SM90 and SM90 execution models, respectively.
