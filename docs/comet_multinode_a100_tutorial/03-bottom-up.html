1–2. Element → token row → expanded row

DIFF Multi-node final accumulation accepts FP16 or BF16. The kernel moves a uint4 pack: 16 bytes, hence eight 16-bit elements. Arithmetic is converted through packed helpers; layer-1 top-k accumulation uses FP32 temporaries before converting back.

token row x[t, 0:H] expanded row x̂[r, 0:H], where r identifies (token t, selected expert j) 0 ≤ t < T, 0 ≤ j < topk, 0 ≤ r < T×topk

scatter_index maps expanded choices to expert-contiguous rows for GEMM0. The inverse routing index makes GEMM1 results addressable again by original token/top-k slot.

3–4. Expert-local problems, M tiles, and N splits

For expert e, layer 0 computes [M_e,H] × [H,F_local] → [M_e,F_local]. Layer 1 computes [M_e,F_local] × [F_local,H] → [M_e,H]. M_e = splits[e] varies, so these are grouped GEMMs.

LayerIndependent overlap dimensionReason
0M / routed rowsRows become legal as each source-rank token shard arrives.
1N / output columnsAn N slice can be gathered and reduced once every expert problem publishes that slice.

CODE V2 gather-RS fixes tile M=128 and tile N=1024; N must be divisible by n_split, and N/n_split must be divisible by 1024. CTA blocks stride through more logical tiles than the GPU can host at once.

5. Convert global ranks into node groups

W = world_size G = nnodes L = W / G # GPUs per node rank = node_idx × L + local_rank

The AG schedule orders ranks relative to the current rank: same-node cyclic local ranks first, then the next node group. Its inverse maps that schedule order back to a global source rank.

src/moe_ag_scatter/sort_util.h:166–188 · HEAD · abridged pseudocodehost/device rank schedule
shift_rank_to_order(rank, dist_env) {
  (node, local) = global_rank_to_node_idx_local_rank(rank);
  order_node  = (node  - dist_env.node_idx  + nnodes) % nnodes;
  order_local = (local - dist_env.local_rank + L) % L;
  return order_node * L + order_local;
}
revert_order_to_rank(order, dist_env) {
  return ((order / L + dist_env.node_idx) % nnodes) * L
       + ((order % L + dist_env.local_rank) % L);
}

Execution. Workspace scheduling uses this order to align M tiles with node-first AG arrival.

Ownership. It is pure topology math; no communication occurs here.

Invariant. The two functions are inverses only under equal, node-contiguous rank groups.

Why. A schedule should consume the shards communication produces earliest.

6. Local peer lists and symmetric slots

V2 local rings receive pointer arrays indexed by local_rank, not global rank:

  • reduce_ptrs[L]: transient local-ring rows;
  • tile_barrier_ptrs[L]: one readiness word per logical tile;
  • barrier[L]: one producer-completion word per N split.

Inter-node storage is symmetric and four-dimensional:

staging_send[G, n_split, staging_rows, n_per] staging_recv[G, n_split, staging_rows, n_per] staging_rows = max_m / topk / W n_per = N / n_split slot(g, sid) = g × n_split + sid slot_base_elements = slot × staging_rows × n_per element_offset = slot_base_elements + local_row × n_per + local_col

Execution. Constructor-time allocation creates capacity-sized storage before any forward call.

Ownership. The operator owns these across calls; only runtime rows within each capacity-sized slot are transferred.

Invariant. max_m/topk and runtime T are divisible by W, and the symmetric heap is large enough on every PE.

Why. Fixed slot strides make sender and receiver address calculations agree while allowing runtime chunks smaller than capacity.

9–10. One local ring for every owner-node group

Execution. Each N split waits for GEMM production, then walks remote owner groups first and this node last.

Ownership / stream. Gather-RS CTAs read node-local peer buffers and write output or symmetric staging on the consumer stream.

Invariant. Every group executes exactly L local stages; remote publication follows all grid blocks.

Why. Produce network-ready node partials early while preserving the V2 local ring.

At each stage, this rank is responsible for segment:

segment(g, stage, local_rank) = g × L + (stage + local_rank + 1) mod L

Stage zero computes this GPU’s local contribution. Intermediate stages add a partial read from this local rank’s reduce buffer and write toward the previous local rank. System-scope tile barriers make the next stage legal.

The last-stage invariant

stage = L − 1 segment = g × L + (L − 1 + local_rank + 1) mod L = g × L + local_rank

That segment is owned by the PE with the same local rank on node g. This is why the host ladder sends to pe = g×L + local_rank.

Execution. A locally owned segment becomes final output immediately; a remotely owned segment is compacted into its owner-node slot.

Ownership. Local output belongs to this rank; symmetric send staging belongs to the sender until RMA completion/reuse boundary.

Invariant. Subtracting global segment row/column origins yields 0≤local_row<runtime_rows and 0≤local_col<n_per.

Why. Convert strided global output coordinates into one contiguous network chunk per (g,sid).

11. The fence → flag → put → signal ladder

  1. Reset. The gather-RS stream zeros every flag/counter and records staging_reset_event. The internode stream waits on it, so a previous call’s flag cannot satisfy the current wait.
  2. Produce. All CTAs write their portion of staging_send[g,sid].
  3. Publish. __threadfence_system(), CTA sync, per-block counter, and last-block system-release store set group_flag[g,sid]=1.
  4. Send. The internode stream waits for that flag then issues a nonblocking putmem_signal to g×L+local_rank.
  5. Name the run. Signal is SET to monotonically increasing run_id, not a reusable binary one.
  6. Receive. Main gather-RS stream waits for every remote source node’s signal for this N split to reach run_id.
  7. Reduce and retire. GPU accumulation adds remote slots. The normal outer wrapper waits for gather-RS completion, then all ranks enter a group barrier before staging reuse.

Execution. Reset, GPU production, flag wait, RMA, signal wait, and accumulation form one ordered ladder.

Ownership / stream. Sender readiness is a CUDA device flag observed by internode_stream; receiver arrival is a symmetric NVSHMEM signal observed on gather_rs_stream.

Invariant. Flag observation orders producer stores; signal observation establishes delivery for its paired put. Run IDs prevent stale receiver success only when all PEs invoke the operator in the same lock-step order.

Why. Preserve GPU-resident dataflow while letting host-enqueued NVSHMEM move coarse contiguous node partials.

12. Reconstruct the Python-visible output

own = local-ring result for segment node_idx×L + local_rank remote[m] = staging_recv[m, sid, 0:runtime_rows, :] output[row, sid×n_per + col] = own[row, sid×n_per + col] + Σ remote[m,row,col] for m != node_idx
src/moe_gather_rs/topk_gather_rs_v2.cu:1021–1056 · DIFF · abridgedGPU final accumulation · FP16/BF16 packs
PackT acc; acc.data = loadPack(output + row*out_n + sid*n_per + col);
for (int m = 0; m < nnodes; m++) {
  if (m == node_idx) continue;
  slot = m*n_split + sid;
  PackT part = loadPack(staging_recv + slot*staging_rows*n_per + row*n_per + col);
  acc.data = addPack(acc, part);
}
storePack(output + row*out_n + sid*n_per + col, acc.data);

Execution. Each launch handles one N split, starting from the own-node contribution already in output.

Ownership. It writes the tensor returned to Python, shape [T/W,N].

Invariant. Every source-node slot has passed its current-run signal wait; packing divides n_per exactly.

Why. Finish cross-node reduction on the owner GPU without another global ring.

Audit checklist

  • CODE nnodes==1 makes L=W, one group, and preserves the old V2 path.
  • DIFF Multi-node forbids do_all_reduce, use_read_mode, and Triton paths.
  • DIFF Layer-1 final accumulation dispatches FP16/BF16 only.
  • DIFF Gather-RS checks runtime m_full≤max_m; AG allocates by max_ntokens but lacks an explicit runtime upper-bound check.
  • CODE Compile-time ceilings are eight local GPUs, 32 world ranks, and eight nodes; V2 AG’s source-segment wait also fits one warp.
  • INFERENCE The normal wrapper’s final collective barrier strongly protects send-staging reuse after all incoming signals. Direct use of the low-level TopkReduceScatterOp.run still deserves a formal NBI source-lifetime audit.
  • INFERENCE Binary AG flags rely on serialized object reuse and a final world barrier; concurrent invocation of one operator object is not documented as safe.

Glossary, snapshot, and sources

Segment
The final token-row shard owned by one global rank.
Owner-node group
The L rank segments whose global ranks share a node index.
Slot stride
The capacity-sized distance between adjacent symmetric staging slots, independent of runtime row count.
